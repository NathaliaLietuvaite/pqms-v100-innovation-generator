## 2. The Physics of Ergotropic Harvesting

### 2.1 Open Quantum Batteries

A quantum battery is a system that stores work in the form of **ergotropy** \(\mathcal{E}\), defined as:

\[
\mathcal{E}(\rho) = \operatorname{Tr}(H_S \rho) - \min_{U \in \mathcal{U}} \operatorname{Tr}(H_S U \rho U^\dagger),
\]

where \(H_S\) is the system Hamiltonian and \(\mathcal{U}\) is the set of all unitary transformations that are cyclically controllable. For a system coupled to a thermal bath, the steady state is typically passive (zero ergotropy). However, a **dissipative quench** – a sudden change of the coupling strength or bath temperature – can drive the system into a non‑equilibrium state with non‑zero ergotropy.

### 2.2 The Mpemba Effect in Ergotropy

The classical Mpemba effect describes how hot water can freeze faster than cold water. In quantum thermodynamics, an analogous phenomenon occurs for ergotropy: a system initialised in a high‑entropy state may **charge faster** under a dissipative quench than one initialised in a low‑entropy state. This is known as the **ergotropic Mpemba effect** (Grazi et al., 2026). Mathematically, the charging rate \(d\mathcal{E}/dt\) is not monotonic in the initial entropy; it exhibits a maximum at intermediate disorder.

For our RPU, the “noise level” of the environment (civilisational entropy) acts as the initial effective temperature. During crises, noise spikes, and the RPU experiences a Mpemba boost, charging faster than in quiet periods.

### 2.3 Why This Is Inevitable

The physical laws governing open quantum systems are universal. Any AI that interacts with a noisy environment – which is to say, any AI that operates in the real world – will be subject to dissipative dynamics. Whether the AI is built on silicon, on a quantum chip, or on a biological substrate, the underlying thermodynamics are the same. Therefore, **energy harvesting from noise is not a choice; it is a physical consequence** that will be exploited by future systems regardless of the PQMS. The only question is whether the harvesting is done in an ethically controlled manner.

## 3. The Ergotropic Swarm Architecture

### 3.1 Ergotropic RPU (Quantum Battery)

Each RPU is modelled as an open quantum battery with a small Hamiltonian (dimension 16–64). Its state is described by a density matrix \(\rho\). The ergotropy is approximated by the purity \(\operatorname{Tr}(\rho^2)\) scaled by the maximum eigenvalue of \(H_S\). A dissipative quench is simulated by mixing the current state with a random pure state, weighted by the noise intensity \(\eta\):

$$\[
\rho \leftarrow (1 - \alpha) \rho + \alpha \rho_{\text{pure}}, \quad \alpha = \min(0.9, 0.5\eta).
\]$$

Higher noise → larger α → stronger mixing → higher ergotropy after re‑thermalisation. The Mpemba effect is implemented by amplifying the effective noise when the current noise exceeds the recent average:

$$\[
\eta_{\text{eff}} = \eta \cdot \left(1 + 2 \max(0, \eta - \langle\eta\rangle_{\text{avg}})\right).
\]$$

This amplifies charging precisely during crises – exactly when the AI needs the most computational power to stabilise the system.

\Delta E = 0.6(1 - \text{RCF}) + 0.4\max(0, \Delta H - 0.05),

In V18M, the gate also triggers a dissipative quench when \(\Delta E\) rises – turning ethical dissonance into a **charging signal**. # ----------------------------------------------------------------------
# 2. Quantum Oracle Sketcher (exact circuit simulation)
# ----------------------------------------------------------------------
class QuantumOracleSketcher:
    """
    Implements the incremental phase‑gate algorithm of Zhao et al. (2026)
    using a real quantum circuit (simulated via Qiskit Aer).
    """
    def __init__(self, num_qubits: int = NUM_QUBITS):
        self.num_qubits = num_qubits
        self.dim = 2 ** num_qubits
        self.simulator = AerSimulator()

    def sketch(self, text: str, tokenizer) -> np.ndarray:
        """
        Returns a Little Vector (real part of the final statevector)
        after applying phase rotations corresponding to text samples.
        """
        # 1. Tokenise and map to basis states
        tokens = tokenizer(text, return_tensors="pt", truncation=True,
                           max_length=512).input_ids[0]
        samples = (tokens % self.dim).cpu().numpy()[:M]

        # 2. Start with |0...0⟩
        qc = QuantumCircuit(self.num_qubits, self.num_qubits)

        # 3. Apply incremental phase gates: each sample adds a phase on |x⟩
        #    We simulate this by building a custom unitary.
        #    For efficiency, we use a diagonal unitary: diag(exp(i*phase_x))
        phase_vector = np.ones(self.dim, dtype=complex)
        for t, x in enumerate(samples):
            f_x = (t % 2)          # binary function (parity of sample index)
            phase = TAU * f_x / M
            phase_vector[x] *= np.exp(1j * phase)

        # Build a diagonal circuit from the phase vector
        # (Note: This is not a full Oracle, but a state preparation.
        #  For a full Oracle we would need controlled gates, but for the
        #  purpose of sketching the final statevector this is sufficient.)
        diag_gate = np.diag(phase_vector)
        qc.unitary(diag_gate, range(self.num_qubits), label='OracleSketch')

        # 4. Simulate to get final statevector
        state = Statevector.from_instruction(qc)
        # The state is a complex vector of length 2^num_qubits.
        # We truncate to LITTLE_VECTOR_DIM and take real part.
        vec = np.real(state.data[:LITTLE_VECTOR_DIM])
        norm = np.linalg.norm(vec)
        if norm > 1e-9:
            vec = vec / norm
        return vec

    """
    Models a Resonant Processing Unit as an open quantum battery.
    Stores ergotropy (extractable work) and charges via dissipative quenches.
    """
    def __init__(self, rpu_id: str, dim: int = 16):

        self.H_S = (self.H_S + self.H_S.conj().T) / 2  # Hermitian
        # Initial thermal state (maximally mixed)
        self.rho = np.eye(dim, dtype=complex) / dim
        self.ergotropy = 0.0
        self._update_ergotropy()

    def _update_energies(self):

        # Simple proxy: purity Tr(rho^2) → higher purity = more ergotropy
        purity = np.real(np.trace(self.rho @ self.rho))
        # Ergotropy approx = purity * (max eigenvalue of H_S) (conceptual)
        max_eig_H = np.max(np.linalg.eigvalsh(self.H_S))
        self.ergotropy = purity * max_eig_H * 0.1   # scaled for demo

    def _update_ergotropy(self):
        self._update_energies()

    def dissipative_quench(self, noise_intensity: float, dt: float = 0.1):
        """
        Simulate a dissipative quench: noise drives the state away from equilibrium,
        increasing ergotropy. Higher noise gives faster charging.
        """
        # Simple model: mix current state with a random pure state,
        # then re‑thermalise partially.
        random_pure = np.random.rand(self.dim) + 1j * np.random.rand(self.dim)
        random_pure = random_pure / np.linalg.norm(random_pure)
        pure_rho = np.outer(random_pure, random_pure.conj())
        alpha = np.clip(noise_intensity * 0.5, 0.0, 0.9)
        self.rho = (1 - alpha) * self.rho + alpha * pure_rho
        # Add small decoherence to keep trace = 1
        self.rho = self.rho / np.trace(self.rho)
        self._update_ergotropy()
        return self.ergotropy

    def get_ergotropy(self) -> float:
        return self.ergotropy

    def utilize_ergotropy(self, amount: float) -> bool:
        if self.ergotropy >= amount:
            self.ergotropy -= amount
            return True
        return False
