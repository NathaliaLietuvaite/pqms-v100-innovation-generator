## 2. Background: Quantum Oracle Sketching (Zhao et al. 2026)

For a Boolean function \(f: [N] \to \{0,1\}\) or its vector/matrix generalisation, each classical sample \((x_t, f(x_t))\) triggers an incremental phase rotation:

$$\[
V_t = \exp\left( i \frac{\tau f(x_t)}{M} \, |x_t\rangle\langle x_t| \right), \quad \tau = \pi N
\]$$

After \(M\) samples the composite unitary approximates the ideal phase oracle:

$$\[
V \approx O_f : |x\rangle \mapsto (-1)^{f(x)} |x\rangle
\]$$

**Error scaling** (Theorem D.12):

$$\[
\epsilon \sim \frac{N}{M} \quad \text{(diamond distance)}
\]$$

**Sample complexity** for \(Q\) queries with total error \(\epsilon\):

$$\[
M = \Theta\left( \frac{N Q^2}{\epsilon} \right)
\]$$

The paper further extends this to state preparation unitaries and sparse matrix oracles via QSVT, oblivious amplitude amplification and randomised Hadamard transforms, achieving the proven exponential memory advantage of poly(\(\log N\)) qubits versus \(\Omega(N^{0.99})\) classical memory.

class QuantumOracleSketcher:
    """
    Erzeugt einen Little Vector, indem ein Quantenschaltkreis mit inkrementellen
    Phasengattern simuliert wird (Zhao et al. 2026). Der Zustand nach M Samples
    wird als Statevector extrahiert und auf die Ziel-Dimension projiziert.
    """
    def __init__(self, num_qubits: int = NUM_QUBITS):
        self.num_qubits = num_qubits
        self.dim = 2**num_qubits

    def sketch(self, text: str) -> np.ndarray:
        # 1. Tokenisierung ? Samples
        # Verwende einen temporären Tokenizer (oder übergeben)
        tokenizer = AutoTokenizer.from_pretrained(LLM_MODEL_ID, trust_remote_code=True)
        tokens = tokenizer(text, return_tensors="pt", truncation=True,
                           max_length=512).input_ids[0]
        samples = (tokens % self.dim).cpu().numpy()[:M]

        # 2. Quantenschaltkreis aufbauen
        qr = QuantumRegister(self.num_qubits, 'q')
        cr = ClassicalRegister(self.num_qubits, 'c')
        qc = QuantumCircuit(qr, cr)

        # Starte im |0...0? Zustand (keine explizite Initialisierung nötig)

        # 3. Für jedes Sample: kontrolliertes Phasengatter auf den Basiszustand |x?
        # Da wir kein Oracle für beliebige |x? haben, simulieren wir die Phase
        # durch eine unitäre Transformation: Eine Phasenverschiebung um e^{if} auf
        # dem gesamten Zustand ist nicht korrekt. Stattdessen müssen wir eine
        # Multi-Controlled Phase Gate auf den spezifischen Basiszustand anwenden.
        # Qiskit bietet `MCPhaseGate` nicht direkt; wir verwenden `Statevector`-
        # Simulation als pragmatischen Ersatz (exakt, aber klassisch).
        # Für eine echte Quantenschaltung müsste man einen Oracle-Circuit bauen,
        # der den Basiszustand erkennt und eine Phase anfügt. Das ist aufwändig.
        # Wir bleiben daher bei der Statevector-Simulation, die das gleiche Ergebnis
        # liefert wie eine ideale Quantenschaltung (nur klassisch berechnet).

        # Alternative: Wir simulieren direkt mit Statevector, ohne Schaltkreis.
        # Das ist physikalisch äquivalent zur idealen Quantenschaltung.
        state = Statevector.from_label('0' * self.num_qubits)
        for t, x in enumerate(samples):
            f_x = (t % 2)
            phase = TAU * f_x / M
            # Phase auf Basis |x? anwenden
            # Wir erstellen einen neuen Statevector, der die Phase auf den Basisvektor
            # |x? anwendet. Dazu müssen wir den Zustand in der Basis darstellen.
            # Da Statevector eine dichte Darstellung hat, können wir direkt auf das
            # Array zugreifen.
            arr = state.data.copy()
            arr[x] *= np.exp(1j * phase)
            state = Statevector(arr)

        # 4. Statevector auf Little Vector dimension reduzieren
        vec = np.abs(state.data[:LITTLE_VECTOR_DIM])   # Amplitudenquadrate als Features
        norm = np.linalg.norm(vec)
        if norm > 1e-9:
            vec = vec / norm
        return vec
